MailgunMessageData

Type declaration

  • [key: string]: any
  • Optional amp-html?: string

    AMP part of the message. Please follow google guidelines to compose and send AMP emails.

  • Optional attachment?: any

    File attachment. You can post multiple attachment values.

    Important: You must use multipart/form-data encoding when sending attachments.

  • Optional bcc?: string

    Same as To but for blind carbon copy

  • Optional cc?: string

    Same as To but for carbon copy

  • Optional from?: string

    Email address for From header

  • Optional h:X-My-Header?: string

    h:' prefix followed by an arbitrary value allows to append a custom MIME header to the message ('X-My-Header' in this case). For example, h:Reply-To to specify Reply-To address.

  • Optional html?: string

    Body of the message. (HTML version)

  • Optional inline?: any

    Attachment with inline disposition. Can be used to send inline images (see example).

    You can post multiple inline values.

  • Optional o:deliverytime?: string

    Desired time of delivery. See Date Format.

    Note: Messages can be scheduled for a maximum of 3 days in the future.

  • Optional o:deliverytime-optimize-period?: string

    Toggles Send Time Optimization (STO) on a per-message basis.

    String should be set to the number of hours in [0-9]+h format, with the minimum being 24h and the maximum being 72h.

    This value defines the time window in which Mailgun will run the optimization algorithm based on prior engagement data of a given recipient. See Sending a message with STO for details.

    Please note that STO is only available on certain plans. See www.mailgun.com/pricing for more info.

  • Optional o:dkim?: boolean | "yes" | "no"

    Enables/disables DKIM signatures on per-message basis. Pass yes, no, true or false

  • Optional o:require-tls?: boolean | "yes" | "no"

    If set to 'True' or 'yes' this requires the message only be sent over a TLS connection. If a TLS connection can not be established, Mailgun will not deliver the message.

    If set to 'False' or 'no', Mailgun will still try and upgrade the connection, but if Mailgun can not, the message will be delivered over a plaintext SMTP connection.

    The default is 'False'.

  • Optional o:skip-verification?: boolean | "yes" | "no"

    If set to True or yes, the certificate and hostname will not be verified when trying to establish a TLS connection and Mailgun will accept any certificate during delivery.

    If set to False or no, Mailgun will verify the certificate and hostname. If either one can not be verified, a TLS connection will not be established.

    The default is False.

  • Optional o:tag?: string | string[]

    Tag string. See Tagging for more information.

  • Optional o:testmode?: boolean | "yes" | "no"

    Enables sending in test mode. Pass yes if needed. See Sending in Test Mode

  • Optional o:time-zone-localize?: string

    Toggles Timezone Optimization (TZO) on a per message basis.

    String should be set to preferred delivery time in HH:mm or hh:mmaa format, where HH:mm is used for 24 hour format without AM/PM and hh:mmaa is used for 12 hour format with AM/PM. See Sending a message with TZO for details.

    Please note that TZO is only available on certain plans. See www.mailgun.com/pricing for more info.

  • Optional o:tracking?: boolean | "yes" | "no"

    Toggles tracking on a per-message basis, see [Tracking Messages](https://documentation.mailgun.com/en/latest/user_manual.html#tracking-messages for details. Pass 'yes', 'no', 'true' or 'false'

  • Optional o:tracking-clicks?: boolean | "yes" | "no" | "htmlonly"

    Toggles clicks tracking on a per-message basis. Has higher priority than domain-level setting. Pass yes, no, true, false or htmlonly.

  • Optional o:tracking-opens?: boolean | "yes" | "no"

    Toggles opens tracking on a per-message basis. Has higher priority than domain-level setting. Pass 'yes' or 'no', 'true' or 'false'

  • Optional recipient-variables?: string

    A valid JSON-encoded dictionary, where key is a plain recipient address and value is a dictionary with variables that can be referenced in the message body. See Batch Sending for more information.

  • Optional subject?: string

    Message subject

  • Optional t:text?: boolean | "yes" | "no"

    Pass yes if you want to have rendered template in the text part of the message in case of template sending

  • Optional t:version?: string

    Use this parameter to send a message to specific version of a template

  • Optional template?: string

    Name of a template stored via template API. See Templates for more information

  • Optional text?: string

    Body of the message. (text version)

  • Optional to?: string | string[]

    Email address of the recipient(s).

    example

    Bob <bob@host.com>. You can use commas to separate multiple recipients.

  • Optional v:my-var?: string

    v: prefix followed by an arbitrary name allows to attach a custom JSON data to the message. See Attaching Data to Messages for more information.
